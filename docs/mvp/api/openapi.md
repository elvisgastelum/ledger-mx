# OpenAPI Specification

OpenAPI 3.0 specification is **generated** from the ts-rest contract using `@ts-rest/open-api`. The single source of truth for API shape is the ts-rest contract defined in `libs/contracts/src/contract.ts`.

## Generation Source

The OpenAPI document is generated by `libs/contracts/src/openapi.ts`, which exports:

- `openApiDocument` - the generated OpenAPI 3.0 document (also exported from `@ledger-mx/contracts/openapi`)
- `writeOpenApiJson(outputPath)` - helper function to write the document to a file

## Regeneration

### Option 1: Using pnpm script (recommended)

From the repository root:

```bash
pnpm --filter @ledger-mx/contracts generate:openapi
```

This writes the artifact to `docs/mvp/api/openapi.json` (relative to repo root).

### Option 2: Using tsx directly

From the repository root:

```bash
pnpm tsx libs/contracts/src/openapi.ts docs/mvp/api/openapi.json
```

### Option 3: Custom output path

```bash
# Relative to repo root
pnpm --filter @ledger-mx/contracts generate:openapi custom/path/openapi.json

# Absolute path
pnpm --filter @ledger-mx/contracts generate:openapi /absolute/path/openapi.json
```

### Option 4: Programmatic usage

```typescript
import { writeOpenApiJson } from "@ledger-mx/contracts/openapi";

// Relative paths resolve from repo root
await writeOpenApiJson("docs/mvp/api/openapi.json");

// Absolute paths work as-is
await writeOpenApiJson("/absolute/path/to/output.json");
```

## Generated Artifact

The static OpenAPI JSON artifact is checked in at `docs/mvp/api/openapi.json`. This file includes:

- All implemented endpoints (auth, category groups, onboarding)
- All planned endpoints (reports, export, health)
- Proper OpenAPI path parameter syntax (e.g., `/category-groups/{id}`)
- Security schemes (bearer JWT authentication)
- Response schemas generated from Zod schemas via `zod-to-json-schema`

## Serving OpenAPI Document

To serve the OpenAPI document with Swagger UI:

### Option 1: Using NestJS Swagger (recommended for development)

```typescript
import { openApiDocument } from "@ledger-mx/contracts/openapi";
// Serve openApiDocument using @nestjs/swagger's setup method
```

### Option 2: Using Swagger UI Express

```typescript
import { openApiDocument } from "@ledger-mx/contracts/openapi";
// Use swagger-ui-express to serve the document
```

## Current Documentation Approach

- Endpoint documentation is maintained in `docs/mvp/api/*.md` files, aligned with the ts-rest contract.
- Schema metadata is added via Zod `describe()` calls in `libs/contracts` schemas.
- No manual `@nestjs/swagger` decorators are used (to avoid diverging from contract source of truth).
